## 3. 오늘(7/19) 확정된 결정 — 전부 docs/decisions.md에 기록됨

1. **공사 우선순위**: ① 디자인(Codex 시안 대기) ② 케어센터 ③ 브리핑 ④ 보장분석 파이프라인 ⑤ 나머지 메뉴
2. **공간 성격**: 브리핑 = 관리·실행실(자료 생성 기능 없음) / 케어센터 = 생성·관리실
3. **기기 역할**: 업무용 노트북 = 주력 1대(개발·발행·고객·보장분석) / 개인 노트북 = 웹앱 사용 전용
4. **보장분석 자료**: 고객에게 절대 전송 안 함. 본인 기기 전용, 원드라이브로 기기 간 이동. 영상 호스팅 불필요
5. **2차 백엔드**: 고객 개인정보 = 네이버 클라우드(NCP) 한국 리전 / 팀 데이터 = Cloudflare 무료 구간.
   **마누스 클라우드는 탈락**(AI 학습 사용 가능성·미국 리전·반출 경로 취약)
6. **세션 분업 지도**: 메뉴가 아니라 작업 성격으로 방을 나눈다. 공용 파일 수정권은 앱 메인 세션만
7. **예약**: 세일즈 코칭 시스템(2차 공사 묶음)

3. 그 세션이 `data/care/issues/weekly-12.json`에 기사·편집장의 말 작성
