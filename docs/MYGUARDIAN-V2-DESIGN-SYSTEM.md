# 마이가디언 v2 디자인 시스템

기준: 디자인 개정 10차 (2026-07-20, 스레드 인텔리전스 언어 이식, 사용자 승인).
구현 파일: `web/assets/platform.css` (플랫폼 공용). 고객 화면(care/card/r)은 별도 편집 문법(`web/assets/style.css`) — 이 문서의 적용 대상이 아니다.

## 1. 디자인 철학

- 금융사·연구기관의 정보 구조를 참고한 한국어 중심 업무 화면.
- 장식보다 정보 계층과 가독성. 밀도는 실데이터로만 만든다 — 가짜 수치 금지.
- 선(1px 테두리)이 그림자를 대신한다. 카드를 띄우지 않고 면을 나눈다.
- 색은 신호로만. 포인트 블루 하나를 규율 있게 반복한다.
- 조용한 모션 — 색만 바뀌고 형태는 움직이지 않는다.
- 사용 주체: 설계사 전용 도구. 편집은 노트북·태블릿, 폰은 열람 중심.

## 3. 타이포그래피

- Pretendard 단일 서체. 위계는 크기·굵기·색으로만.
- 전역: 본문 자간 `-0.02em`, 행간 1.6, `word-break: keep-all`, 수치 `tabular-nums`.
- 위계: 페이지 제목 `clamp(24px, 2.4vw, 29px)`/700/자간 -0.04em → 섹션 제목 18px/750/-0.035em
  → 본문 13px → 보조 라벨 11px → eyebrow 10px/800/자간 +0.1em.
- 핵심 수치 27px 이상 굵게.
- 화면 최상단은 eyebrow → 제목 → 설명의 3단 헤딩.

## 4. 레이아웃

- 앱 셸: 고정 사이드바 240px + 콘텐츠. 콘텐츠 폭 `min(1180px, 100% - 60px)`.
- 상단바 62px sticky + `backdrop-filter: blur(8px)`, 흰 반투명(0.97) + 하단 1px 선.
- 여백 예산은 바깥에: 페이지 패딩 30px, 섹션 사이 28px, 카드 패딩 16~21px. 내부는 촘촘하게.
- 검색·목록형 화면은 2단 목록·상세(`.workspace-grid`).

## 5. 사이드바

- `#102a4e` 다크 네이비 + 상단 시안 4px(`border-top`). 라이트 콘텐츠와 2-톤 대비.
- 메뉴 2그룹: "업무"(홈·고객관리·청구·지급·사례·판례·학습·브리핑·우리팀) /
  "발행"(케어센터·카톡카드). 그룹 라벨 `.nav-group-label` 9px/자간 +0.14em.
- 항목: 43px, radius 4px, 흰 72% 텍스트. hover = 흰 5.5% 배경.
  활성 = 흰 10% 배경 + 좌측 2px 시안 막대(`#43c8df`).
- 하단 `.sidebar-status` 화면별 안내문. admin 미노출, 메뉴 상한 9개.

## 6. 컴포넌트

- 카드(`.surface-card`·`.action-card`): 흰 표면 + 1px 선 + radius 6px + 극미세 그림자.
  hover는 배경 미세 틴트(`#fafbfd`)와 테두리 강조만 — 리프트·확대 금지.
- 지표 격자(`.stat-row`): gap 0 + 셀 사이 1px 칸막이 선. 표처럼 붙인다. 모바일은 세로 칸막이.
- 배지(`.badge-*`·`.dday-chip`): radius 3px, 연한 배경 + 진한 동색 텍스트.
- 탭(`.tab-button`): 활성 = 채운 배경 pill(`--platform-blue-soft` + 진파랑 텍스트), 밑줄 없음.
- 선택 표시: 좌측 3px 인셋 막대(`box-shadow: inset 3px 0`) — 목록 선택(`.case-row.active`) 공통.
- 버튼: 주액션 = 파랑 배경+흰 텍스트, 보조 = 흰 배경+회색 선. radius 4px. 그림자 없음.
- 입력: radius 4px, 테두리 `--platform-line-strong`, 포커스 = 파란 테두리 +
  `0 0 0 2px rgba(7,88,214,.09)` 링.
- 강조 패널: 상단 3px 포인트 컬러 스트립.

## 7. 상태 표현

- 상태색은 의미 고정: 지급/완료=그린, 부지급/오류=레드, 일부지급/주의=앰버, 중립=회색.
- 색만으로 전달 금지 — 텍스트를 항상 병기한다.
- 빈 상태(`.empty-inline`): "현재 상태(strong) + 다음 행동 안내(span) + 필요 시 이동 링크(a)".
  "데이터 없음"만 쓰지 않는다. 가짜 CTA는 만들지 않는다.
- 신뢰도 등급(검증완료/부분검증/미검증-참고용)·사건번호·고정 고지 문구는 데이터 신뢰성 원칙.

## 8. 모션

- 0.14초 색 전환(색·배경·테두리)만. 확대·눌림·리프트·등장 애니메이션 금지.
- 진행바 등 상태 표시만 0.3초 허용. `prefers-reduced-motion` 존중.

## 9. 반응형

- 검증 3해상도: 1920 / 1180(태블릿) / 390(모바일). `scripts/screenshot.py`.
- 1100px: 사이드바 224px, 콘텐츠 폭 축소.
- 767px: 사이드바 → 오프캔버스(햄버거), 상단바 60px, 그리드 1열, 터치 타겟 44px.
- 케어·카톡카드는 모바일 퍼스트 유지 (고객 화면 예외).

## 10. 접근성

- `:focus-visible` 2px 파란 아웃라인 유지. 키보드 접근 보장.
- 텍스트 대비 4.5:1 이상(faint 색상은 이 기준으로 고정).
- 터치 타겟 44px (모바일 미디어쿼리에서 확보).

## 11. 바로가기 설치 (PWA)

- `web/install.html`: 설치 상태 배지, 상황별 주버튼(프롬프트/iOS 안내/완료),
  기기별 3열 안내, 주소 복사 폴백. 홈 사이드바 버튼이 진입점 (앱 실행 중에만 숨김).
- `web/sw.js`: 앱 셸 캐시 `myguardian-shell-v1`. navigate·JSON = 네트워크 우선 + 캐시 폴백,
  정적 자산 = 캐시 우선. 버전명 교체로 갱신. 외부 CDN·비 GET은 캐시하지 않는다.
