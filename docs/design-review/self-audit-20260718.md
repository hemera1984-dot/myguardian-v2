- 스키마: cases.json 726건 ↔ case.schema.json, issues.json 15건 ↔ care-issue.schema.json — jsonschema 검증 전건 통과
- 상호작용: 검색·필터·상세 선택·스와이프·카톡 복사 자동 테스트 통과, 콘솔 오류 0
- 3해상도(1920/1180/390) 스크린샷 깨짐 없음

## 발견·즉시 수정한 문제

1. 케어 이미지 61.2MB → 8.4MB 최적화 (최대 폭 1200px + JPEG 재압축, pipeline/optimize_images.py 공정화). 모바일 고객이 기사당 최대 1.7MB를 받던 문제
2. 확장자 위장 SVG 2장 (weekly-10 표지, weekly-11 4면 삽화): .jpg로 저장돼 브라우저에서 깨지던 상태 → 래스터화로 복구
3. 서가 표지 inline 레이아웃 겹침 (카톡 버튼 클릭 가로채임) — 전 커밋에서 수정

## 미해결 — 개선 과제 (우선순위순)

1. 사례 결과 태깅: decision_result '기타' 621건 — 결과 필터의 실효성이 낮다. tag_cases.py로 요약문 근거 재분류 필요 (근거 없으면 미분류 유지)
2. 검색 품질: 현재 단순 부분일치. 공백·대소문자 처리만 있고 초성 검색, 동의어(암/악성신생물), 다중 키워드 AND가 없다. 홈 통합검색(사례+용어+케이스카드)도 미구현
3. 접근성: 사례·고객 목록 행이 div — 키보드 선택 불가. role/tabindex 또는 button화 필요. 스와이프 리더에 스크린리더 페이지 안내 없음
4. 코드 중복: esc()·사이드바 토글이 페이지마다 복붙 6곳 — web/assets/common.js로 추출할 시점 (패치 누적 전에)
5. cases.json 505KB 통짜 로드 — GitHub Pages gzip으로 당장은 허용 범위. 1,500건 넘으면 목록용 축소본/상세 분리 필요
6. 고객관리 읽기 전용 — 신규 등록·메모 저장(File System Access 쓰기) 미구현
7. 다크모드 변수만 존재, 전환 스위치 없음 (헌법상 후순위 — 유지)
8. 케어 카톡 문구 계절 인사가 고정 문장 — 발행 공정에서 월별 갱신하거나 작성함에서 수동 수정 (현행)
