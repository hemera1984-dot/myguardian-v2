코드를 붙이지 않고 언어만 추출해 헌법을 개정한 뒤 전 화면에 일관 적용한다.

## 스레드 인텔리전스 (2026-07-19, 사용자+GPT 5.6 Sol 제작)

개인 자료 검증센터. 스크린샷 4장(기기 설치·수집함·검증·수집 설정) 기준.

### 채택 후보 — 마이가디언에 이식할 가치가 있는 문법

1. **사이드바 카운트 뱃지** — 메뉴 항목 우측에 실데이터 건수(수집함 2, 검증 7).
   마이가디언 적용: 사례·판례 383, 학습 693, 케어센터 발행 호수 등. 밀도 원칙과 정확히 일치.
2. **사이드바 하단 시스템 상태판** — 상태 점(초록/노랑) + 항목별 한 줄 상태.
   마이가디언 적용: 데이터 검증 현황(검증완료 209 등), 발행 D-day, 고객 저장소 연결 상태.
3. **다크 네이비 사이드바 + 라이트 본문** — 잉크 네이비(#1a2744 계열) 사이드바가 화면의
   중심축을 잡아준다. 현재 마이가디언은 화이트 사이드바 — 전환 여부는 Codex 시안과 함께 판단.
4. **상단바 상시 통합검색** — 모든 화면 상단에 검색 + 단축키 힌트(/) + 주요 액션 버튼.
   마이가디언은 홈에만 통합검색이 있다 — 전 화면 상시화 검토.
5. **킥커+대수치 헤드라인** — 작은 라벨(자간 넓힘) 위, 큰 수치 제목("오늘 새 자료 1건").
   영문 장식 금지이므로 한글 킥커("오늘의 브리핑")로 변환 적용.
6. **스탯 행의 서브캡션** — 수치 아래 작은 설명("검토 대기 0건", "반대 의견 확인").
   마이가디언 자료 현황 타일에 이미 유사 문법 — 서브캡션을 더 서사적으로.
7. **점수 시각화** — 큰 수치(86/100) + 세부 항목 가는 진행바(관련성 94·신뢰성 84).
   마이가디언 적용: 사례 신뢰도 등급의 시각화 후보 (등급 배지 → 배지+바).
8. **단계 스트립** — 1→2→3 공정을 화살표로 잇는 가로 스트립 (1차 검토→반대 검토→최종 결정).
   마이가디언 적용: 발행 데스크 3단계, 청구 도구의 절차 안내.
9. **판정 비교 카드** — 두 검토 의견을 좌우 카드로 대치시키고 하단에 선택 버튼.
   마이가디언 적용: 추후 브리핑·코칭, 사례 상세의 쟁점 대비 표시.
10. **번호 매긴 설정 카드 + 상태 뱃지** — 01/02 큰 회색 번호, 우상단 인증 대기 뱃지,
    하단 메타(일정·담당). 절제된 카드 위계의 좋은 예.
11. **좌측 액센트 보더 강조 카드** — 파란 왼쪽 테두리로 "지금 할 일" 카드를 구분.
    마이가디언 발행 데스크의 초안 카드가 이미 유사(앰버 보더) — 문법 일관화.

### 거리를 둘 것

- 영문 킥커(TODAY'S RESEARCH BRIEF)는 헌법의 영문 장식 금지와 충돌 — 한글로 치환해 수용.
- 상단바의 버튼 2개+상태 표시는 스레드인텔의 업무 흐름에 맞춘 것 — 마이가디언은 화면별
  주 액션이 달라 일괄 이식하지 않는다.

### 후속

- 사이드바 네이비 전환·상시 검색은 Codex 디자인 시안 검토와 묶어 한 번에 판단 (우선순위 1번).
