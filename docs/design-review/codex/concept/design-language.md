# Codex 디자인 시안 — 플랫폼 디자인 언어

## 목적

이 시안은 `web/design-concept/`에 격리된 디자인 검토용 대표 화면이다. 기존 기능·데이터·운영 화면을 수정하지 않는다. 사용자 승인 후 Claude Code가 아래 디자인 언어를 추출해 헌법을 개정하고 전체 화면에 다시 적용하는 것을 전제로 한다.

## 핵심 방향

- 홍보형 랜딩 화면이 아니라 매일 사용하는 보험 업무 대시보드로 시작한다.
- 첫 화면의 우선순위는 통합검색 → 오늘의 업무 → 실데이터 현황 → 주요 업무 진입 → 최근 발행물이다.
- 수치와 이력은 저장소 실데이터만 사용한다. 고객 수, 매출, 일정 등 근거 없는 수치를 만들지 않는다.
- 마이가디언과 스레드 인텔리전스는 같은 제품군의 프레임을 공유하되 목적에 따라 정보 구조를 달리한다.
- 네이비는 업무 프레임, 블루는 실행·선택, 청록은 브랜드 식별선, 상태색은 판정 신호로만 사용한다.

## 구성 문법

1. 사이드바는 9개 메뉴를 한 번에 보여주며, 활성 메뉴는 면과 2px 식별선으로 표시한다.
2. 상단바는 현재 위치와 최소한의 전역 동작만 둔다.
3. 페이지 제목 아래에 강한 통합검색을 배치해 모든 업무의 첫 진입점으로 사용한다.
4. 오늘 처리할 업무는 얇은 우선 업무 스트립으로 표시한다. 장식용 배너는 사용하지 않는다.
5. 수치 카드는 한 묶음으로 연결해 숫자 비교가 빠르게 되게 한다.
6. 업무 바로가기는 번호·제목·한 줄 설명·동작으로 구성한다.
7. 케어센터 발행물은 고객 콘텐츠이므로 실제 표지를 사용하되 플랫폼 카드 안에서 절제해 표시한다.
8. 표와 목록은 헤어라인으로 구분하고 불필요한 그림자·배경색을 사용하지 않는다.

## 반응형 원칙

- 1180px: 사이드바와 업무 밀도를 유지하고 보조 라벨만 줄인다.
- 900px 이하: 업무 패널을 1단으로 전환한다.
- 720px 이하: 사이드바를 햄버거로 전환하고 지표를 2열로 배치한다.
- 모바일에서도 통합검색과 오늘의 업무가 첫 화면 안에 들어와야 한다.

## Claude Code 수용 지침

- 시안의 HTML·CSS를 운영 화면에 그대로 복사하지 않는다.
- 사용자 승인 후 색·타이포·간격·구성 문법을 `CLAUDE.md`와 `AGENTS.md`의 디자인 조항으로 먼저 정리한다.
- 공용 토큰과 셸부터 적용한 뒤 홈 → 고객관리 → 청구·지급 → 사례·판례 → 학습 → 브리핑 → 우리팀 순으로 확장한다.
- 케어센터·카톡카드·고객 리포트는 고객용 편집 문법을 유지하고 플랫폼 셸만 같은 제품군으로 맞춘다.
- 로고 6종 중 최종안이 정해지면 현재 임시 심벌 자리만 교체한다.
