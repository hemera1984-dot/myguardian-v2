## 2. 재검수 대상 — findings-03 각 항목의 해소 확인 (재현 우선)

### [치명1] 미팅 차수 XSS
- `web/customers/index.html`의 목록 배지·상세 배지·aria-label 세 경로에서 `차수`가
  `esc(String())`를 거치는가. 악성 프로필(`"미팅":[{"차수":"<img src=x onerror=...>",...}]`,
  또한 `상태`·`일자`에도 페이로드)로 목록·상세 렌더 시 스크립트가 실행되지 않는지 재현하라.
  차수 외 다른 미팅 필드(상태 클래스 조합 포함) 주입 경로도 함께 확인하라.

### [중요1] 프로필 동시 쓰기 무결성
- `queueProfileWrite`·`commitProfile`이 같은 고객 profile.json 쓰기를 직렬화하는가.
  진행상태·미팅 완료·미팅 일자 변경을 쓰기 완료 전에 빠르게 연속 실행할 때, 최종 파일과
  화면(`c.profile`)이 일치하고 실패 원복이 다른 성공 쓰기를 되돌리지 않는지 재현하라.
  복제-교체(성공 후에만 `c.profile` 교체)가 실제로 적용됐는지 확인.

### [중요2] 서비스워커
- `web/sw.js` v3에서 이동·JSON의 4xx/5xx 응답이 정상 캐시를 덮지 않고 폴백하는가(`cacheable`
  게이트). 자산 백그라운드 갱신이 `event.waitUntil()`로 완료 보장되는가. 구버전→v3 전환 시
  새 HTML+옛 CSS 혼합이 남지 않는지, 오프라인 폴백이 유지되는지 확인하라.

### [중요3] 단건 점검 baseline 보존
- `pipeline/monitor_insurers.py --code <코드>` 실행 시 나머지 자동 회사 지문이 보존되는가.
  알 수 없는 코드·오타·접속실패 케이스에서 상태 파일이 손실되지 않는지 재현하라.

### [중요4] 모바일 접근성
- 닫힌 사이드바가 모바일(≤767px)에서 `inert`·`aria-hidden`으로 탭 순서·스크린리더에서
  제외되는가. 열 때 첫 메뉴 포커스·Escape/오버레이 닫기·햄버거 복귀가 동작하는가.
  홈·케어 서재·카톡카드·설치·관리자의 주 조작 요소가 모바일에서 44px 이상인지 재측정하라.

### [중요5] 케어 서재·데스크 디자인
- 발행 데스크 D-day 칩·버튼·입력, 서재 최신호 카드·표지·카톡 버튼에 개정 10차 이전 색
  (#2563eb)·radius(999/9/10px)·확대/리프트/등장(mg-rise) 모션이 남아 있는지 계산 스타일로
  재확인하라. manifest·홈 theme-color가 확정 팔레트(#102a4e/#f5f7fa)인지. 잡지 리더·
  카톡카드의 네이비·골드·섹션색은 보존됐는지.

### [중요6] 폐기 도구 잔존
- 홈 청구·지급 카드 문구, 청구 검색 접근성 이름, current-status.md에 폐기된 "알릴의무·
  지급사례 검색" 표현이 남아 있는지. 실제 화면(분쟁 사례 자료실·12유형·보험사 감시)과 일치하는지.

### [제안] 링크 스킴
- `window.mg.safeUrl()`이 청구 사례 원문·실손24·보험사 안내·케어 데스크 뉴스 링크에
  적용돼 `javascript:`·`data:` 스킴이 무력화되는지 확인.

## 3. 회귀 확인
