## 치명

### [치명] 1. 미팅 차수 저장형 XSS — 수용·수정
- `web/customers/index.html`의 목록 배지·상세 배지·aria-label 3경로에서 `차수`를

## 중요

### [중요] 1. 프로필 동시 쓰기 무결성 — 수용·수정
- 고객코드별 Promise 체인(`queueProfileWrite`)으로 같은 profile.json 쓰기를 직렬화.
- `commitProfile`: 현재 객체가 아니라 복제본에 변경을 적용하고, 파일 커밋 성공 후에만
  `c.profile`을 교체. 실패는 메모리를 손대지 않아 다른 성공 쓰기를 되돌리지 않는다.
- 진행상태·미팅 3경로 모두 이 경로를 공유. 직렬화 순서 보장을 테스트로 확인(역순 지연 입력도 입력순 완료).

### [중요] 2. 서비스워커 오류 캐시·갱신 미보장 — 수용·수정
- `web/sw.js` v3: `cacheable()` 게이트로 2xx·basic 응답만 저장. 이동·JSON은 비정상 HTTP
  응답을 캐시하지 않고 기존 정상 캐시로 폴백. 자산 백그라운드 갱신은 `event.waitUntil()`로
  완료 보장, `cache.put()` 완료를 await. 캐시명 v2→v3로 구버전 정리.

### [중요] 3. 단건 점검이 baseline 삭제 — 이미 수정(검수 중 반영분)
- `pipeline/monitor_insurers.py`: `--code` 부분 실행 시 미점검 회사는 이전 지문 보존,
  요약은 최종 항목에서 집계. (findings에 "미커밋 수정 감지"로 참고된 그 수정이며 커밋 완료.)

### [중요] 4. 모바일 사이드바 탭 순서·44px — 수용·수정
- `web/assets/platform.js`: 닫힌 사이드바(모바일)에 `inert`+`aria-hidden`. 열 때 첫 메뉴로
  포커스, Escape·오버레이 닫기 시 햄버거로 복귀. 데스크톱은 항상 접근 가능.
- 44px 보정: 홈 프로필 버튼·설치 링크·관리자 입력/버튼(platform.css), 케어 채널탭·서재 카톡·
  발행 버튼·카톡카드 저장(style.css)에 모바일 min-height 44px.

### [중요] 5. 케어 서재·데스크 옛 색·모서리·모션 잔존 — 수용·수정
- `style.css`: 하드코딩 `#2563eb`→`var(--accent)`, pub-chip·서재 카톡·입력 radius→3/4px,
  latest 카드·표지 확대·서가 리프트·`mg-rise` 등장·`scale(0.97)` 눌림 모션 제거(색·그림자 변화만).
- `manifest.webmanifest`·홈 `theme-color`를 확정 팔레트(`#102a4e`/`#f5f7fa`)로.
- 보존: 잡지 리더(mag-*)·카톡카드 딥네이비+골드는 그대로.

### [중요] 6. 폐기된 판정 도구 잔존 안내 — 수용·수정
