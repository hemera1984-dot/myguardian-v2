# 4차 검수 대응 — 전 건 수정 완료, 재심 요청

발견 8건(치명 1·중요 6·사소 1) 전부 수정했다. 검증은 Codex의 재현 방법을 그대로 따라
자동 재검증 20건으로 확인했다 (모형 FSA 핸들, cp949 기본 콘솔, 저장형 주입 등).

## 조치 내역

### [치명 1] 이관 검증 없는 원본 삭제 → 복사·검증·확정 3단계로 재시공
- 복사 전 원본 매니페스트(상대 경로·크기·SHA-256) 생성 → 복사 → 대상 파일을 전건 재열람해
  경로·크기·해시 대조 → 전건 일치 시에만 "원본 삭제 확정" 버튼 노출 (고객코드·파일 수 재표시)
- 확정 전에는 어떤 경로로도 원본을 삭제하지 않는다. 취소 시 대상 사본을 제거해 되돌린다
- 검증·복사 실패 시 원본 유지 + 불완전 사본의 위치와 수동 정리 필요 여부를 문구로 명시
- 재검증: 확정 단계 표시·확정 전 원본 보존·확정 후 사본 내용 일치·취소 되돌리기 각 통과

### [중요 2] 자기복사 허용 → 연결 저장소 트리 전체 차단 + 대상 저장소 검사
- isSameEntry 재귀 대조(containsHandle)로 대상이 연결 루트 트리 안(원본 자신·하위·형제·루트)이면 거부
- 대상이 고객 저장소로 보이는지 검사: mg-clients* 이름 / 빈 폴더 / C-코드 폴더 보유 중 하나
- 재검증: Codex 재현(대상=고객 폴더 자체) 차단 + 원본 무손상, 비저장소 대상 차단 통과

### [중요 3] 본문 자료형 미검증 → care-body.schema.json 신설 + 검증기 강화
- publish_care_issue.py가 검증: body.id==메타 id, 채널별 기사 수(주간 4·월간 3),
  번호 1~N 연속·유일, 제목·부제 비공백 문자열, 요약·한마디·편집장의말 문자열 배열,
  본문 {t:h|p, x:비공백} 블록 배열, 이미지·커버 실존
- 재검증: 문자열 본문·id 불일치·기사 수 오류·번호 중복 각 차단, 정상 본문 발행 성공 통과

### [중요 4] cp949 콘솔 오류 → 출력 인코딩 명시 + 원자적 쓰기
- 파이프라인 3종 모두 시작 시 stdout/stderr를 utf-8(errors=replace)로 재구성
- issues.json·본문 쓰기는 임시 파일 작성 후 os.replace 원자 교체 — 절반 쓰인 파일 불가
- 재검증: cp949 기본 콘솔에서 실패·성공 경로 모두 예외 없이 목록/완료 문구 출력

### [중요 5] 프로필 사진 속성 주입 → 형식 화이트리스트
- 사진 값은 `data:image/(jpeg|png);base64,` + base64 문자만 + 길이 상한을 통과해야 렌더
  (따옴표·공백이 정규식상 불가능해 속성 탈출 자체가 성립하지 않음). 불합격 값은 로드 시 폐기
- localStorage 용량 초과 시 사진만 버리고 텍스트 프로필을 지키며 안내 문구 표시
- 재검증: Codex 재현 페이로드 주입 → onerror 미실행, 이니셜 자리표시로 복구

### [중요 6] 카운트업 중간값 → 제거
- 수치는 로드 즉시 실값 표시. 가짜 수치 금지 원칙 위반을 인정하고 수치 모션 자체를 제거
- 재검증: 로드 직후 383 정확 표시

### [중요 7] 키 안내 구 개발자센터 → NAVER API HUB로 통일
- 안내 문구를 KEY_GUIDE 상수 한곳으로 모으고 console.ncloud.com 경로 + 구 키 비호환 경고 명시
- 재검증: 키 없는 환경 실행 시 새 안내 출력

### [사소 1] 모바일 고정 배경 → scroll 전환
- 720px 이하에서 background-attachment: scroll. 재검증: 390px 계산값 scroll 확인

## 재심 요청 범위
- 이관 실패·성공·취소 경로 (특히 검증 단계의 우회 가능성)
- 유효·무효 발행물 (care-body 규격 경계값)
- 나머지는 4차 통과 항목 유지 — 회귀만 확인 요청
